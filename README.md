# Binary Zenith Visual Tests

This project provides automated visual regression testing using Playwright. It also includes a crawler script to dynamically extract a list of URLs to test across a target website.

## Prerequisites

- Node.js
- npm

## Installation

Install the required dependencies:

```bash
npm install
```

## Configuration

Set up your `.env` file by copying the provided example:

```bash
cp .env.example .env
```

Ensure you configure the `PROD_URL` and `STAGING_URL` variables in the `.env` file based on your environments.

## Available Commands

### 1. Crawl URLs

Discover URLs across your site and generate a baseline list of paths to test:

```bash
npm run crawl -- --url=https://your-site.com --depth=2 --limit=50
```
- `--url`: The target base URL. If omitted, uses `PROD_URL` or `STAGING_URL` from `.env`.
- `--depth`: How many levels deep to crawl (default: 1).
- `--limit`: Maximum number of paths to collect (default: 100).
- `--selector`: *(optional)* A CSS selector to scope link extraction to a specific DOM element. When provided, only `<a>` tags inside the matching element are harvested and added directly to the path list — no recursive crawling occurs. Useful for grabbing just the top navigation or sidebar links.

**Selector example** — collect only URLs from a top nav menu:
```bash
npm run crawl -- --url=https://your-site.com --selector="nav.top-menu"
```

The output paths are saved neatly in `data/paths.json`.

### 2. Update Baseline Snapshots

To generate the initial "golden" snapshots against your production environment:

```bash
npm run test:update-baseline
```

### 3. Run Comparison Tests

To run the visual tests against your staging environment and compare them with the baseline:

```bash
npm run test:compare
```

### 4. View Test Report

If any visual tests fail, Playwright generates a comprehensive diff report. You can view it by running:

```bash
npm run report
```

## Directory Structure
- `scripts/`: Contains utility scripts, including `crawl.js`.
- `tests/`: Contains the Playwright test specifications (`visual.spec.js`) and authentication setup.
- `data/`: Autogenerated. Contains the baseline paths (`paths.json`) tested by Playwright.
